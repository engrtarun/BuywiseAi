================================================================================
BuyWise AI - Comprehensive Project Overview & Codebase Documentation
================================================================================

This document details the current state, architecture, database schemas, features, 
potential improvement areas, and known issues of the BuyWise AI Next.js + Supabase application,
based directly on a codebase analysis of src/app, src/components, src/lib, src/hooks, 
and database access paths.

────────────────────────────────────────────────────────────────────────────────
1. FULL FEATURE LIST
────────────────────────────────────────────────────────────────────────────────

AUTHENTICATION & ONBOARDING
* Email/Password Authentication: Complete email and password sign-up and login flow 
  in `app/signup` and `app/login`, with strength checklist validation on signup 
  (minimum 8 characters, at least 1 digit, and at least 1 uppercase letter).
* Google OAuth: Native OAuth sign-in integration on both login and signup screens, 
  which routes exchange callbacks securely through `app/auth/callback/route.ts`.
* Email OTP Verification: 6-digit OTP email confirmation powered by Supabase Auth. 
  It automatically intercepts logins for unverified accounts and prompts for code 
  entry, featuring a client-side 30-second resend cooldown timer.
* Password Reset: Reset link requests sent via `supabase.auth.resetPasswordForEmail` 
  which routes users back to `/welcome` or profile editing on validation.
* Route Protection Middleware: Secure route interceptor in `lib/supabase/middleware.ts` 
  that checks active sessions and guest status. Unauthenticated users are redirected 
  to `/login` when trying to access `/`, `/chat`, or `/welcome`.
* Onboarding Wizard: First-time Google OAuth signups are redirected to `/welcome` 
  to capture the user's full name, upserting it into the database `profiles` table.

CHAT INTERFACE & EXPERIENCE (GEMINI AI)
* AI Chat with Gemini: Real-time conversational interface inside the chat shell 
  using the `@google/generative-ai` SDK and the fast, lightweight `gemini-2.5-flash` model.
* Context-Aware History: Map frontend message structures (`role: assistant | user`) 
  into the Gemini SDK roles (`role: model | user`) in `/api/chat/route.ts`.
* Daily Message Limit with IST Reset: Daily throttle of 25 messages for authenticated users. 
  Counters in `daily_message_limits` are stamped and refreshed relative to the 
  Indian Standard Time (IST) offset (+5.5 hours, offset-calculated in server actions) 
  so the message allowance resets exactly at midnight IST.
* Local Fallback Responses: In-memory fallback library (`lib/fallbackResponses.ts`) 
  that serves detailed mock recommendations if Gemini API calls fail, timeout, or 
  hit 429 quota limits.
* Persistent Chat History: Messages and sessions are loaded on mount from 
  `chat_messages` and `chat_sessions` database tables.
* Auto-Generated Chat Titles: Sending the first message triggers an asynchronous 
  background call to Gemini to summarize the text into a concise 3-6 word chat title, 
  syncing it back to the database. Falls back to character-truncation if the API fails.
* Chat Management: Rename (inline pencil button) and delete operations in the 
  sidebar directly sync with database session fields.
* Resizable Sidebar Drawer: Resizable sidebar panel using a custom `useSidebarResize` 
  drag hook, collapsible into a space-saving icon-only view for smaller viewports.

THEMING
* BuyWise Green / Dark Switcher: Sleek, responsive theme presets declared as custom 
  Tailwind v4 tokens in `app/globals.css`. Supports "BuyWise Green" (emerald ink & paper) 
  and "BuyWise Dark" (monochrome carbon & neutral light).
* LocalStorage Persistence: User theme choice is stored locally as `buywise-theme` 
  and synced on window loading via `ThemeInitializer.tsx` using `data-theme` 
  on the document element.
* Application Modes: Toggles between "retail" (e.g. fashion/tech) and "food" modes. 
  In Food mode, the primary brand accent changes to Swiggy Orange (`#FC8019`) and 
  swiping features switch to culinary menus.

QUICK BUY (DECK SWIPING)
* Live Product Fetching: Curated listings fetched dynamically from FakeStoreAPI and 
  formatted into INR currency (multiplied by 85) with simulated available sizes.
* Profile-Based Filters: Products are filtered dynamically by user size and budget 
  pulled from profile preferences.
* Stack Pagination & Prefetching: Products are shown in a Tinder-style stack. 
  When the visible deck falls below 3 cards, the app pre-fetches the next page 
  from the API route.
* Swipe Actions: Swipe right to save to the cart/bookmarks (persisted to Supabase 
  `user_saved_products`), and swipe left to reject. Uses haptic vibration and 
  audio sound cues (`accept.wav` / `reject.wav`).
* Saved Deck Cart: Display screen for saved cart items. Supports updating quantities, 
  deleting, and displaying totals with mock tax (18%) and shipping.
* Order Logging (One-Click Buy): Swipe checkout button log order data immediately 
  into the database `orders` table.

PROFILE MANAGEMENT
* Completion Progress Ring: Circular SVG progress ring (circumference of 106.8) 
  rendered in the sidebar around the user's avatar, tracking profile completeness 
  based on full name, avatar, size, and budget.
* Avatar Upload to Supabase Storage: File selection, client-side type validation (size 
  and image type), and uploading via Supabase Storage SDK. Uploads to a public 
  `avatars` bucket under `${userId}/${timestamp}.${ext}`, automatically attempting 
  to initialize the bucket if missing.
* Preference Customization: Edit profile screen supporting direct updates of full name, 
  preferred sizing (S, M, L, XL), and max budget (INR).

VIRTUAL WARDROBE & COLLAGE CANVAS
* Virtual Wardrobe (`/virtual-wardrobe`): Dedicated studio page with a toggle control 
  to switch between Mix & Match and Outfit Canvas views.
* Mix & Match (Mix Mode): Splits saved wardrobe products into separate top and bottom 
  motion panels. Supports dragging or paginating through tops/bottoms, with a 
  randomized "Mix" shuffle button.
* Outfit Canvas (Canvas Mode): Multi-item drag-and-drop workspace powered by 
  `@hello-pangea/dnd`. Move items from the bottom source dock onto the canvas. 
  Supports "free collage" or "grid" layout modes.
* Draggable Wardrobe Item: Draggable card handles with responsive 3D tilt effects 
  during active drag movement, driven by Framer Motion velocity tracking.
* Outfit Rating (Match Meter): Interactive rating system. Sends canvas items list to 
  `/api/outfit-rating` which hits Gemini to return a witty fashion review 
  and a style score (0-100). The Match Meter animates the score with a circular 
  SVG loader and slot-machine overshoot effect.
* Canvas Thinking Wave: Glowing overlay with an animated five-bar neon wave indicating 
  Gemini is reviewing the fit.
* Quick Access Menu: Plus button on `ChatInput` opens a React Portal popover 
  providing quick links to Quick Buy and Virtual Wardrobe.
* Floating Chat Bubble: Fixed button on Quick Buy and Wardrobe pages that redirects 
  back to `/chat`.

DATABASE SCHEMA (SUPABASE TABLES)
1. `profiles`
   * `id`: uuid (Primary Key, references auth.users.id)
   * `full_name`: text (Nullable)
   * `email`: text (Nullable)
   * `avatar_url`: text (Nullable)
   * `size`: text (Default 'M')
   * `budget`: numeric (Default 5000)
   * `sizes`: text[] or comma-separated text (Nullable)
   * `max_budget`: numeric (Nullable)
   * `preferred_sizes`: text[] (Nullable)
   * `preferred_categories`: text[] (Nullable)
   * `preferences`: jsonb (Nullable)
2. `chat_sessions`
   * `id`: uuid (Primary Key, Default: gen_random_uuid())
   * `user_id`: uuid (references auth.users.id / profiles.id)
   * `status`: text (Default: 'active')
   * `requirements`: jsonb (Default: '{}')
   * `title`: text (Nullable)
   * `created_at`: timestamptz (Default: now())
3. `chat_messages`
   * `id`: uuid (Primary Key, Default: gen_random_uuid())
   * `session_id`: uuid (references chat_sessions.id ON DELETE CASCADE)
   * `sender`: text (Value constraints: 'user' or 'ai')
   * `message`: text (Content markdown)
   * `created_at`: timestamptz (Default: now())
4. `orders`
   * `id`: uuid/bigint (Primary Key)
   * `user_id`: uuid (references auth.users.id)
   * `product`: jsonb (Stores object: { id, name, price, image_url })
   * `payment_method`: text (Default: 'Quick Buy (Swipe)')
   * `status`: text (Default: 'completed')
   * `created_at`: timestamptz (Default: now())
5. `user_saved_products`
   * `id`: uuid/bigint (Primary Key)
   * `user_id`: uuid (references auth.users.id)
   * `product_id`: text
   * `product_name`: text
   * `price`: numeric
   * `image_url`: text
   * `is_cart`: boolean
   * `created_at`: timestamptz (Default: now())
6. `daily_message_limits`
   * `user_id`: uuid (references auth.users.id)
   * `usage_date`: date/text (Format: YYYY-MM-DD)
   * `message_count`: integer (Default: 1)
   * Note: Composite Primary Key on `(user_id, usage_date)`
7. Storage Bucket
   * `avatars` (Public bucket containing profile avatars path format: `${user_id}/${timestamp}.${ext}`)

2. 50 IMPROVEMENT / REVIEW POINTS
────────────────────────────────────────────────────────────────────────────────

SECURITY
1. Review RLS policies on `daily_message_limits` to ensure users cannot manipulate 
   their own message counts or insert records for dates in the future.
2. Remove client-side calls to `supabase.storage.listBuckets()` and `createBucket` 
   in `profile/page.tsx`, which require high-privilege keys, and instead configure 
   buckets using Supabase SQL migrations.
3. Add input sanitization on incoming markdown in `MessageBubble.tsx` to prevent XSS 
   vulnerabilities when utilizing `rehype-raw` with user or AI-generated inputs.
4. Implement token-based authentication checks inside the `/api/outfit-rating` 
   POST endpoint to prevent unauthenticated access and potential API abuse.
5. Require active session verification inside `/api/quick-buy` to avoid public 
   scraping of external api routes.
6. Verify and validate the product price server-side in `/api/quick-buy/actions` 
   before inserting records into `orders` or `user_saved_products` to block client-side 
   payload modification (e.g. changing price to 0).
7. Restrict image mimetype uploads to strict types (e.g., only `image/png`, 
   `image/jpeg`) on the Supabase storage bucket policies, reinforcing the client 
   file extension validation.
8. Enforce rate limiting on OTP request endpoints (e.g. `resend`) to mitigate email 
   spam abuse vectors.
9. Implement a strict content security policy (CSP) header configuration in 
   `next.config.ts` to restrict execution to trusted sources only.
10. Ensure Postgres Row-Level Security (RLS) is enabled on all tables including 
    `orders` and `user_saved_products`, double checking policies using `auth.uid()`.

GEMINI FREE TIER LIMITATIONS & RETRIES
* High-frequency requests triggers 429 quota exceptions (rate limits). To mitigate 
  this, the system implements fallback responses and a custom message cooldown delay, 
  but does not guarantee AI responses if key limitations are exceeded.

UNTESTED & UNVERIFIED EXPERIMENTAL FEATURES
* Outfit Canvas, Match Meter, and `/api/outfit-rating` fashion commentary endpoint 
  have been merged but lack automated testing coverage. The influencer commentary parsing 
  relies on a fragile regex match for the format `SCORE: XX`, which is vulnerable to 
  Gemini response format variations.

DATABASE & NAMING INCONSISTENCIES
* Sizing Columns: The database and code alternate between `size` (text field in `profiles`), 
  `sizes` (comma-separated string array in API route fallbacks), and `preferred_sizes` 
  or `preferences` properties.
* Budget Columns: Sizing forms, profiles, and API routers alternate between checking 
  `budget`, `max_budget`, and camel-cased `maxBudget`.
* Image Columns: Card lists use `image` for URL properties, while orders and saved 
  products records store the URL under `image_url`, necessitating runtime mapping code: 
  `image_url: product.image_url || product.image`.
* Storage Bucket Listing permission check: The frontend profile page attempts to run 
  storage bucket list queries directly. This fails with permission violations in 
  production environments where client keys lack admin policies.
